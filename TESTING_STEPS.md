# 決賽 Prototype 測試步驟

最後更新：2026-06-30

本文件給評審、PM 與 QA 直接操作「從碳客變捷客」概念原型。原型不需要登入、不需要網路 API，也不會送出真實會員、票務、付款或區塊鏈資料。

## 測試入口

- 線上 prototype：<https://sofiayan0523.github.io/from-carbon-to-metro/>
- 本地 server：在 repo 根目錄執行 `python3 -m http.server 4173 --bind 127.0.0.1`，再開啟 <http://127.0.0.1:4173/>
- 離線包：`offline/from-carbon-to-metro-offline.zip`
- 建議瀏覽器：Chrome、Edge 或 Safari 的近代版本
- 狀態重置：重新整理頁面即可回到初始 demo state

注意：在開發分支尚未合併到 `main` 前，GitHub Pages production URL 仍可能顯示舊版。Phase 6 branch QA 請先用本地 server 或離線包驗證；送件前需合併到 `main` 並重新跑本文件的線上測試流程。

## 測試前提

- 主流程完全使用本機 mock data 與 deterministic JavaScript。
- AI 推薦不呼叫外部模型或商家 API；排序分數由本機商家資料與固定權重計算。
- 碳足跡公式固定為 `90 gCO2e/km x 通勤距離`。
- 綠點加成門檻固定為本月 5 kg -> `1.2x`、10 kg -> `1.5x`。
- Numbers proof 使用固定示範月份 `2026 / 06` 與 demo hash，目的是讓離線 QA、截圖與現場 demo 可重播；正式產品會改用真實事件時間與真實事件 hash。
- Proof payload 只呈現月度統計與區間 bucket，不呈現完整起訖站、商家、精確時間或個人識別碼。

## A. 線上測試流程

1. 開啟線上 prototype。
   - 預期看到頁首「從碳客變捷客」與左側四個導覽卡：`今天`、`路線`、`減碳`、`我的`。
   - 右側手機框底部也有同樣四個 tab。
   - 預設位於 `今天`，畫面顯示「6 月 30 日 週二」、「早安，Sofia」與捷運點 `1,240`。

2. 在 `今天` 點選 `完成今天通勤 · 領取回饋`。
   - 預期按鈕文字變為 `今天通勤已記錄 ✓`。
   - 預期本月減碳由 `8.2 kg` 更新為 `8.9 kg`。
   - 預期捷運點數由 `1,240` 動畫更新為約 `1,258`。
   - 預期今日通勤顯示 `0.72 kg` 與 `+18 點`。
   - 點選任一張「綠點卡」後，預期顯示「綠點 +10 / +30 / +15 已入帳」其中一種結果。

3. 點選 `路線`。
   - 預期看到板南線站點 `忠孝復興`、`忠孝敦化`、`國父紀念館`、`市政府`、`永春`。
   - 預期看到 `今日路線任務`，集滿 3 枚綠章可獲 `+50 綠點`。
   - 點選 `餐飲`、`購物` 或 `生活` 篩選後，預期商家清單依分類收斂。
   - 點選 `在「誠品生活」集章 · +5` 或當前選取店家的集章按鈕，預期集章數增加；集滿 3 枚後顯示 `路線任務完成！`。

4. 點選 `減碳`。
   - 預期看到日 / 月 / 年切換、`本月累積減碳`、5 kg / 10 kg 門檻與 14 天趨勢圖。
   - 點選 `日`、`月`、`年`，預期主數字與說明切換。
   - 點選 `+` 增加額外通勤，預期本月減碳與進度條同步更新；達到 10 kg 後綠點倍率更新為 `1.5×`。
   - 點選 `−`，預期額外通勤數不會低於 `0`。

5. 點選 `我的`。
   - 預期看到成就徽章牆、`Numbers Protocol` 可信證明區塊與 opt-in 控制。
   - opt-in 開啟時，預期 proof record 顯示 `2026 / 06`、`減碳量（去識別化）`、`距離區間 40–60 km`、`任務類型 每日通勤` 與 demo hash `0x7f3a·b8e2·c91e`。
   - 點選 opt-in switch 關閉後，預期 proof record 隱藏；再點一次可恢復。
   - 點選 `重播稽核軌跡`，預期顯示 `稽核軌跡已重播：3 筆月度統計 hash 一致 ✓`。
   - Proof 畫面不應顯示完整起訖站、商家、精確時間、精確通勤距離或個人識別碼。

6. 在 `我的` 點選 `分享`。
   - 預期出現「分享減碳成果」dialog。
   - 預期 dialog 顯示本月減碳、捷運點與 demo hash。
   - 按 `Esc`、右上角 `x` 或 `完成` 可關閉。

## B. 離線測試流程

1. 將 `offline/from-carbon-to-metro-offline.zip` 複製到沒有網路的電腦。

2. 解壓縮後進入 `from-carbon-to-metro/` 資料夾。

3. 優先用本地 server 開啟：

   ```bash
   python3 -m http.server 4173 --bind 127.0.0.1
   ```

   開啟：

   ```text
   http://127.0.0.1:4173/
   ```

4. 如果現場電腦沒有 Python，直接用瀏覽器開啟解壓後的 `index.html` 作為 fallback。
   - Demo 邏輯仍可執行。
   - 若特定瀏覽器限制 `file://` 字型載入，畫面會回退到系統中文字型，不影響主流程判讀。

5. 依照「A. 線上測試流程」第 2 到第 6 步重跑一次。
   - 預期結果應與線上流程一致。
   - 原型不應出現等待 API、登入、付款、外部圖片、區塊鏈 RPC 或 Capture / Numbers live API 的阻塞狀態。

## C. 桌面與手機版檢查

1. 桌面寬度測試。
   - 使用一般筆電或桌機瀏覽器開啟。
   - 左側導覽卡與右側手機框不應互相重疊。
   - 手機框內容可捲動，底部導覽列不應遮住主要按鈕。
   - 分享 dialog 開啟後不應溢出畫面。

2. 手機寬度測試。
   - 使用瀏覽器 responsive mode，建議寬度 390 px 或 420 px。
   - 四個導覽卡、手機框、底部導覽列不應水平溢出。
   - Proof record 的長 payload 可換行，不應撐破卡片。

## D. 素材與隱私檢查

- Repo 內保留本地字型與授權文件於 `assets/fonts/`，新版 bundle 另使用本地固定版本 runtime 於 `assets/vendor/`。
- 原型沒有使用官方捷運 Logo、未授權圖片、音樂、地圖底圖或第三方品牌素材。
- 商家名稱只作為文字 mock 情境，不含商標圖、廣告圖或交易資料。
- Proof payload 只保留月份、減碳值、距離區間、任務類型、同意狀態與 demo hash。
- 主流程沒有 secrets、tokens、API key、真實會員資料、真實票務紀錄或付款資料。

## E. 通過標準

- 線上 URL 可完成今天 -> 路線 -> 減碳 -> 我的 -> 分享卡主流程。
- 離線包在無網路環境可完成同一主流程。
- `我的` 頁在 opt-in 後可重現 demo hash `0x7f3a·b8e2·c91e`。
- 任何 live integration 文案都只能是加分展示，不得阻塞主 demo。
- 桌面與手機寬度下沒有主要文字、按鈕、卡片或底部導覽列互相覆蓋。
